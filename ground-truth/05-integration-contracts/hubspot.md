# Integration Contract — HubSpot (+ Slack)

This is a **must-keep** behaviour of the current site and a core capability of the new one: site
forms feed HubSpot, prospects **book calls directly**, and the team is **notified in Slack** in
real time. Losing any of these on the rebuild is a regression.

## Systems & roles
- **HubSpot** — system of record for **contacts, leads, meetings, pipeline**. Provides the **forms**
  and the **Meetings scheduler** (`/contact-hubspot-scheduler`) that lets prospects book a call directly.
- **Slack** — **notification only** (not a source of truth). Posts to the sales channel on a new
  form submission and on a booked call.
- **Website** — captures intent and hands off; never a shadow CRM.

## Flows
1. **Form submit:** site → HubSpot contact upsert (idempotent by email) with UTM/source/persona signal.
2. **Notify:** HubSpot → **Slack** message to the sales channel ("new lead: …").
3. **Direct scheduling:** prospect books via HubSpot Meetings → meeting created + calendar invite
   + **Slack** booking notification.

## Source of truth per concern
| Concern | Source of truth | Notes |
|---------|-----------------|-------|
| Contact / lead | **HubSpot** | site writes; reads only to avoid dupes |
| Meeting / booking | **HubSpot Meetings** | calendar invite generated by HubSpot |
| Pipeline / stage | **HubSpot** | sales-owned |
| Slack message | Slack | derived event; safe to lose without data loss |

## Auth & security
- HubSpot access via **server-side** token / private app — never in client code or URLs.
- Slack via HubSpot's native Slack integration (or a server-side webhook).
- **No PII in URLs/query strings** (privacy rule). Honor cookie/consent before non-essential tracking.

## Failure / retry
- HubSpot write fails → **durable queue + idempotent retry by email**; the lead is never lost; UI degrades gracefully.
- **Slack notification fails → MUST NOT block** lead creation or booking; retry/log out of band.
- Scheduler unavailable → still capture the lead; offer async follow-up.

## Guardrails
- BR-8: Slack messages contain lead info only — never internal revenue/deal-size/anti-ICP data.
- Scheduler embed must be **accessible** (keyboard + screen reader) and not tank Core Web Vitals.

## Open questions
- Exact HubSpot **portal ID / form IDs / Meetings link** and the target **Slack channel** (→ Sales; OQ-1 owner).
- Consent/cookie handling for HubSpot tracking.
- Keep HubSpot-native scheduler embed vs. a custom front-end calling HubSpot APIs.
