# Explorer backend — deploy runbook

How to take the Ground Truth Explorer from the local single file to a deployed, **internal-only**
app with a **live "Ask the Ground Truth" chat**. Implements Decisions 0005 (internal-only Claude
chat), 0016 (deploys to Vercel) and 0017 (this architecture). Keep secrets server-side only.

## What runs where

| Piece | Where | Notes |
|-------|-------|-------|
| Explorer UI (`ground-truth-explorer.html`) | Vercel static (`public/index.html`) | Generated by the build from Git. Contains GT content → must sit behind access control. |
| Chat API (`/api/ask`) | Vercel serverless function (Node) | Retrieves over the Git-rebuilt index, calls the Anthropic API, returns answer + citations. |
| Retrieval index (`api/_gt-index.json`) | Generated by the build | "The index, rebuilt from Git" (Decision 0010). Not served statically. |
| Auth (optional but recommended) | Supabase | Per-user sign-in + email-domain restriction; the function checks the session JWT. |

Nothing else (no database, no pgvector) is required for this slice — see Decision 0017 for why
that is deferred.

## Environment variables (Vercel project → Settings → Environment Variables)

**Server-side only (never exposed to the browser):**

- `ANTHROPIC_API_KEY` — required for live answers. Without it, chat degrades to retrieval-only
  (it still returns cited sources).
- `SUPABASE_URL` — your Supabase project URL (also used by the build; it is public).
- `SUPABASE_ANON_KEY` — Supabase anon/publishable key (public).
- `ALLOWED_EMAIL_DOMAIN` — e.g. `rootstrap.com`. The function rejects sessions whose email is
  outside this domain. Optional.
- `REQUIRE_AUTH` — `true` to force the JWT check even if you haven't set `SUPABASE_*`. Optional;
  it is implied whenever `SUPABASE_URL` + `SUPABASE_ANON_KEY` are present.
- `GT_ASK_MODEL` — optional model override (default `claude-sonnet-4-6`).
- `GITHUB_TOKEN` — required to **decide** (write a decision outcome back to Git). Use a fine-grained
  PAT scoped to this repo with **Contents: read/write**. `/api/decide` commits the outcome to the
  decision's `.md`. Without it, deciding falls back to a manual-commit message (nothing breaks).
- `GITHUB_REPO` / `GITHUB_BRANCH` — optional; default `anthonyfig/rs-site-gt` / `main`.

`SUPABASE_URL` and `SUPABASE_ANON_KEY` are read **at build time** and embedded in the page so the
browser can sign users in. They are public values by design. The Anthropic key and the Supabase
service-role key are **never** read by the build and never reach the client.

## 1. Internal-only (do this first)

The served HTML contains Ground Truth content, so the deployment itself must be gated:

- Turn on **Vercel Deployment Protection** for the project (Settings → Deployment Protection →
  Standard Protection / password / SSO). This is the actual gate for the whole app.
- The Supabase sign-in and the `/api/ask` JWT check are **defense in depth** on top of that — they
  identify the user and keep chat answers from reaching anyone outside `ALLOWED_EMAIL_DOMAIN`.

## 2. Supabase (recommended)

1. Create a Supabase project. Copy the project URL and the anon key into the Vercel env vars above.
2. Auth → Providers → Email: enable **magic link / OTP**.
3. Restrict who can sign in to internal users — e.g. invite-only, or an email-domain allow-list, and
   keep `ALLOWED_EMAIL_DOMAIN=rootstrap.com` so the API rejects anything else.
4. Auth → URL configuration: add your Vercel domain to the allowed redirect URLs.

If you skip Supabase, the app still deploys and chat still works — rely on Vercel Deployment
Protection for internal-only, and the sign-in card is simply not shown.

## 3. Deploy to Vercel

1. Import `rs-site-gt` as a Vercel project (root directory = `rs-site-gt`).
2. Framework preset: **Other**. `vercel.json` already sets the build command
   (`node tools/ground-truth/build-explorer.mjs`) and output (`public/`).
3. Set the environment variables, then deploy. The build regenerates the HTML and
   `api/_gt-index.json` from Git on every deploy (the index is always fresh).

## 4. Local development

```bash
npm run gt:explorer     # build HTML + public/index.html + api/_gt-index.json
# open ground-truth-explorer.html directly → runs in "local mode":
#   browse + decisions workspace work; chat answers in retrieval-only (cited sources, no model call)

# to run the live function locally (build first so /api/ask has its index):
npm run gt:explorer && npx vercel dev   # needs ANTHROPIC_API_KEY in the environment
```

The CLI version of the same retrieval + answer path is `tools/ground-truth/ask.mjs`
(`node tools/ground-truth/ask.mjs "your question"`), handy for testing retrieval without the UI.

## Security checklist

- [ ] Deployment Protection is on (HTML is not anonymously reachable).
- [ ] `ANTHROPIC_API_KEY` and the Supabase service-role key exist only in server env, never in the
      build or the client. (The build only reads `SUPABASE_URL` / `SUPABASE_ANON_KEY` / `EXPLORER_API_BASE` / `ALLOWED_EMAIL_DOMAIN`.)
- [ ] `ALLOWED_EMAIL_DOMAIN` is set so `/api/ask` refuses non-Rootstrap sessions.
- [ ] `api/_gt-index.json` and `public/` are git-ignored (rebuilt on deploy).

## Deferred (Decision 0017)

Supabase Postgres + **pgvector** retrieval, the validate→commit write path (US-5), and the full
a11y/eval pass. Add pgvector when the corpus or query volume makes keyword retrieval insufficient.
